# Product Platforms & Reference Architectures

This directory contains the product-level architectures and technical reference architectures for each major CyberCom platform service.

## Core Product Reference Architectures

*   **[CyIdentity Reference Architecture](cyidentity_reference_architecture.md):** OAuth 2.1 authentication flow, identity realms, and MFA passkey layouts.
*   **[CyIntegrationHub Reference Architecture](cyintegrationhub_reference_architecture.md):** HL7 ADT MLLP socket endpoints, SMART on FHIR, and ZATCA compliance mapping.
*   **[CyData Reference Architecture](cydata_reference_architecture.md):** Spark ETL processing, Apache Iceberg table layouts, and PII anonymization gates.
*   **[CyAI Reference Architecture](cyai_reference_architecture.md):** Triton ML model serving, SaMD medical validation, and CDS Hooks endpoints.
*   **[CyMed Reference Architecture](cymed_reference_architecture.md):** Clinical EHR engine reference, break-glass flows, and competitive analysis against Epic, Cerner, TrakCare, and Hakeem.
*   **[CyCom Reference Architecture](cycom_reference_architecture.md):** Financial general ledger, procurement workflows, segregation of duties, and competitive analysis against SAP, Odoo, Oracle ERP, and Dynamics 365.
*   **[CyShop Reference Architecture](cyshop_reference_architecture.md):** Retail catalog, shopping carts, credit card payment gateways, and fulfillment pipelines.
*   **[CyGov Reference Architecture](cygov_reference_architecture.md):** Public registry systems, land records, permitting workflows, and government cloud deployment guidelines.
*   **[CyConnect Reference Architecture](cyconnect_reference_architecture.md):** Notification routers (SMS, email, push notifications) and secure clinician chat.
*   **[CyCitizen Reference Architecture](cycitizen_reference_architecture.md):** Citizen portal, mobile identity wallet, and national e-ID connections.
