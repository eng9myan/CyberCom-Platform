# Requirements

> Functional, non-functional, regulatory, and stakeholder requirements for every CyberCom product.

This folder was created during **Program 0, Phase 0.2 (Governance)**.
Content will be authored progressively under the governance defined in [../../CONTRIBUTING.md](../../CONTRIBUTING.md) and [../governance/repository_operating_model.md](../governance/repository_operating_model.md).

## Scope

- **Functional requirements** — what each product must do.
- **Non-functional requirements (NFRs)** — performance, scalability, availability, security, accessibility, observability.
- **Regulatory requirements** — HIPAA, GDPR, ISO 27001, SOC 2, MOH, e-government baselines.
- **Stakeholder requirements** — by persona (clinician, finance officer, citizen, admin).

## Conventions

- One requirement per atomic statement, with a stable ID: `REQ-<area>-NNNN`.
- Each requirement links to at least one ADR or design doc once implemented.
- Use [EARS](https://alistairmavin.com/ears/) syntax for behavioral requirements when possible.
