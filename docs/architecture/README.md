# Enterprise Reference Architectures

This directory contains the formal reference architectures of the CyberCom Platform, defining structural, network, security, and database guidelines across all deployment tiers.

## Index

*   **[Enterprise Reference Architecture](enterprise_reference_architecture.md):** The master consolidation framework linking all platform layers.
*   **[Logical Reference Architecture](logical_reference_architecture.md):** Tier layering, service boundaries, and event-driven routing rules.
*   **[Physical Reference Architecture](physical_reference_architecture.md):** Network layouts, DMZ configuration, VMs, and cluster specifications.
*   **[Database Reference Architecture](database_reference_architecture.md):** PostgreSQL conventions, UUIDv7 indexing, and partitioning strategies.
*   **[Security Reference Architecture](security_reference_architecture.md):** Zero Trust flow, ABAC controls, and emergency break-glass protocols.
*   **[Integration Reference Architecture](integration_reference_architecture.md):** Healthcare standards (FHIR, HL7, DICOM), EDI, and e-invoicing.
*   **[Deployment Reference Architecture](deployment_reference_architecture.md):** SaaS, private cloud, sovereign cloud, and hybrid deployment profiles.
*   **[Observability Reference Architecture](observability_reference_architecture.md):** OpenTelemetry logging, metrics collection, tracing, and SLA/SLO definitions.
*   **[Multi-Tenant Reference Architecture](multi_tenant_reference_architecture.md):** Compute and database-level isolation policies.
*   **[Master Data Reference Architecture](master_data_reference_architecture.md):** System of Record (SoR) mappings and golden record reconciliation.
*   **[Document Management Reference Architecture](document_management_reference_architecture.md):** Digital signatures, security zones, and JCI retention schedules.
*   **[Environment Reference Architecture](reference_environment_architecture.md):** Configurations and sizing for Dev, Test, UAT, Stage, and Prod environments.
