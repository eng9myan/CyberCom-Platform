# Standards

> Engineering, security, data, API, UX, and documentation standards that all CyberCom code must follow.

This folder was created during **Program 0, Phase 0.2 (Governance)**.
Content will be authored progressively under the governance defined in [../../CONTRIBUTING.md](../../CONTRIBUTING.md) and [../governance/repository_operating_model.md](../governance/repository_operating_model.md).

## Scope

- **Coding standards** per language (TypeScript, Python, Java, Kotlin, Swift, Go).
- **API standards** — REST/OpenAPI, GraphQL, gRPC, event schemas.
- **Data standards** — naming, modelling, classification, retention.
- **Security standards** — crypto, authn/authz, secrets, logging, PHI/PII handling.
- **UX & accessibility standards** — WCAG 2.2 AA baseline, design tokens.
- **Documentation standards** — Markdown, diagrams, ADRs.

## Conventions

- Standards are **normative**. Deviations require an ADR.
- Each standard names an owner and a review cadence.
