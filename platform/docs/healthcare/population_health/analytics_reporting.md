# Population Health Analytics & National Reporting

## Purpose

Analytics provides dashboards, AI-generated insights, KPI tracking, and outbreak forecasting. Reporting manages the lifecycle of national health reports from drafting through government submission.

## Analytics Models

### NationalHealthSnapshot
A point-in-time summary of population health state: registered patients, disease prevalence by code, vaccination coverage by vaccine, care gap rate, risk distribution, active outbreak count, and quality score. `unique_together` on (tenant, snapshot_date, period_type, geographic_scope). Snapshots are generated by a scheduled background task — never manually created in production.

### PopulationAnalyticsInsight
AI-generated or algorithmically-detected insight about a pattern in the population. `insight_type`: disease_trend / care_gap / risk_pattern / program_effectiveness / outbreak_risk / vaccination_gap / quality_gap / population_shift. All AI insights: `is_advisory_only=True` (non-editable). Workflow: pending_review → acknowledged → acted_upon / dismissed.

### QualityKPIDashboard
Facility or population-level KPI tracking with trend direction (improving/stable/declining). `kpi_category`: clinical_quality / patient_safety / access / efficiency / patient_experience / population_health.

### OutbreakForecast
CyAI-generated disease outbreak prediction. `is_advisory_only=True` (non-editable). Includes predicted case count and confidence interval for a specified `forecast_period_days`. A public health officer must acknowledge before any operational response is initiated.

### PopulationHealthDashboard
Snapshot roll-up for a specific view (population_health / registry / public_health / national / ministry / surveillance): counts of enrolled patients, high-risk patients, open care gaps, active programs, vaccination rate, registry count, and KPI summary JSON.

## Reporting Models

### NationalReport
Formal report output. `report_type`: annual_health / disease_surveillance / registry_report / vaccination_coverage / quality_report / outbreak_report / program_report / ministry_report / who_report. Lifecycle: draft → in_review → approved (requires `approved_by_user_id`) → submitted → published. `fhir_measure_report_id` references the FHIR MeasureReport resource.

### ReportTemplate
Defines the structure, sections, and fields for each report type. `is_national_standard=True` marks templates conforming to MOH or WHO standards.

### GovernmentSubmission
Tracks the submission of an approved report to a government authority. `submission_method`: electronic / api / email / portal / manual. `reference_number` is the acknowledgement reference from the receiving authority. Status: pending → submitted → acknowledged / rejected / accepted.

### ReportSchedule
Automated report generation schedule. `auto_generate=True` triggers report creation at `next_due_date`. `auto_submit=False` by default — automatic submission to authorities requires explicit opt-in and approval at the GovernmentSubmission level.

## API Endpoints — Analytics

| Method | Path | Description |
|---|---|---|
| GET | `/analytics/snapshots/` | National health snapshots |
| GET | `/analytics/insights/` | Population insights |
| POST | `/analytics/insights/{id}/acknowledge/` | Acknowledge AI insight |
| GET/POST | `/analytics/quality-kpis/` | Quality KPI dashboard |
| GET | `/analytics/outbreak-forecasts/` | CyAI outbreak forecasts (advisory) |
| GET/POST | `/analytics/dashboards/` | Population health dashboards |

## API Endpoints — Reporting

| Method | Path | Description |
|---|---|---|
| GET/POST | `/reporting/reports/` | Create / list reports |
| POST | `/reporting/reports/{id}/approve/` | Approve for submission |
| POST | `/reporting/reports/{id}/submit/` | Submit to authority |
| GET/POST | `/reporting/templates/` | Report templates |
| GET/POST | `/reporting/submissions/` | Government submission records |
| GET/POST | `/reporting/schedules/` | Report generation schedules |

## AI Guardrails

- All `OutbreakForecast` records: `is_advisory_only=True, editable=False`
- All `PopulationAnalyticsInsight` records: `is_advisory_only=True, editable=False`
- `NationalReport.status` cannot be set to "submitted" without `approved_by_user_id`
- `GovernmentSubmission` requires `submitted_by_user_id` (human actor always required)
