# Workflows

> End-to-end business and operational workflows (user journeys, BPMN-style flows).

This folder was created during **Program 0, Phase 0.2 (Governance)**.
Content will be authored progressively under the governance defined in [../../CONTRIBUTING.md](../../CONTRIBUTING.md) and [../governance/repository_operating_model.md](../governance/repository_operating_model.md).

## Scope

- **Business workflows** — admit-discharge-transfer, procure-to-pay, hire-to-retire, citizen onboarding, …
- **Operational workflows** — incident response, release, on-call escalation.
- **Cross-product workflows** — flows that span two or more CyberCom products.

## Conventions

- Use Mermaid `sequenceDiagram` or `flowchart`; BPMN `.bpmn` files allowed alongside.
- Each workflow names actors, triggers, pre/post conditions, error paths, and SLAs.
